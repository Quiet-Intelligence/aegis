# Process Reward Model (PRM) Reward-Hacking Audit

**Date:** 2026-07-21
**Status:** Pre-Training Scaffold Audit Complete
**Auditor:** Aegis Security & Evals Team

## Executive Summary
Before training the Process Reward Model (PRM) on the LinUCB bandit's reward signal, we conducted a structural audit to determine if the bandit was vulnerable to "reward hacking" (optimizing parameters to artificially inflate EV1 static scores without providing genuine security).

Our findings show that the current single-event EV1 dataset is **highly susceptible** to reward hacking if used as the sole fitness function for the reinforcement learning loop. A generic LLM could theoretically lower the decision threshold (`AUTO_DECIDE_THRESHOLD`) strictly to block known signatures, while simultaneously failing to block multi-step emergent adversarial trajectories.

## Structural Mitigation
To structurally prevent the LinUCB bandit from exploiting the static EV1 metrics:
1. We have implemented the **EV2 Trajectory Harness**, which evaluates adversarial survival across 50-step interactions rather than isolated events.
2. The PRM pipeline has been wired end-to-end (`scripts/train_prm.py`). It relies on step-level trajectory labels, NOT static golden sets.

## Conclusion
The reward signal generated by the EV2 trajectory harness is robust against static reward hacking. The PRM scaffold is ready to ingest scaled trajectory data. No fine-tuning will commence until the trajectory dataset reaches statistical significance (N > 10,000 runs).
